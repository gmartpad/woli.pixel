# 08 — better-auth Integration for Woli Pixel

**Date:** 2026-04-04
**Status:** Approved

## Problem

All API routes are currently public. There is no user authentication, session management, or access control. Routes that cost money (AI API calls) or mutate user-owned data are exposed without protection.

## Solution

Integrate better-auth with email/password + username + Google OAuth authentication. Use Resend for transactional emails (verification, password reset). Protect mutation/cost routes behind session middleware.

## Authentication Methods

- **Email/password** — registration with email verification via Resend
- **Username plugin** — users can sign in by username instead of email
- **Google OAuth** — social login with automatic account linking

## Backend (`apps/api`)

### New Files

- `src/auth.ts` — better-auth instance with Drizzle adapter, email/password, username plugin, Google OAuth, Resend callbacks, session config
- `src/db/auth-schema.ts` — generated by `npx auth@latest generate` (4 core tables + username columns)
- `src/middleware/auth.ts` — Hono middleware: calls `auth.api.getSession()`, sets `c.var.user` / `c.var.session`

### Modified Files

- `src/index.ts` — mount auth handler on `/api/auth/*`, apply auth middleware to protected routes, update CORS for credentials
- `src/db/schema.ts` — import and re-export auth schema
- `.env.example` — add auth env vars

### Route Protection

| Status | Routes |
|--------|--------|
| Public | `/health`, `/image-types`, `/generation-cost`, `/api/auth/*` |
| Protected | `/images`, `/batches`, `/generate`, `/audits`, `/brands`, `/quality-gates` |

### Session Strategy

Cookie-based database sessions (better-auth default). Cookie caching enabled (5 min, compact strategy) to reduce DB load.

## Frontend (`apps/web`)

### New Files

- `src/lib/auth-client.ts` — `createAuthClient` with username plugin
- `src/components/LoginPage.tsx` — email/password + username form, Google OAuth button
- `src/components/RegisterPage.tsx` — registration form (name, email, username, password)
- `src/components/AuthGuard.tsx` — redirects to login if no session

### Modified Files

- `src/App.tsx` — add login/register views, wrap protected tabs with AuthGuard, replace hardcoded "GM" avatar with session user
- `src/lib/api.ts` — add `credentials: "include"` to all fetch calls

## Email (Resend)

- **Verification** — sent on sign-up via `emailVerification.sendVerificationEmail` callback
- **Password reset** — sent via `emailAndPassword.sendResetPassword` callback
- Sender: `RESEND_FROM_EMAIL` env var

## Database Tables

| Table | Key Columns |
|-------|------------|
| `user` | id, name, email, emailVerified, image, username, displayUsername, createdAt, updatedAt |
| `session` | id, token, userId, expiresAt, ipAddress, userAgent |
| `account` | id, userId, providerId, accountId, accessToken, refreshToken, password |
| `verification` | id, identifier, value (token), expiresAt |

## Environment Variables

```
BETTER_AUTH_SECRET=<generated>
BETTER_AUTH_URL=http://localhost:3000

GOOGLE_CLIENT_SECRET=...

RESEND_FROM_EMAIL=noreply@yourdomain.com
```
